Tank Wars

by: Anay

This game was made for my level 2 project at The League of Amazing Programmers.
Click here to download and play!

1. Why I Made This Game

I made this game because I used to play a similar game with friends and wanted to see if I could recreate it.

2. What I Learned

I learnt that its pretty hard to try and dodge bullets and attack the opponent at the same time. Some feedback I got was that the tanks were sometimes able to go off screen so I added some features to help minimize that problem.

3. Architecture of My Game

The GameObject class is the parent class for the tank, projectile, and power up classes, creating the basic elements of all game objects including collision boxes, x and y positions, size, etc. The GamePanel class handles the GUI of the game which is the menu, game, and end screens as well as a visual scoreboard, while also handling the controls. The ObjectManager class handles the tracking and spawning of power ups and projectiles, and updating the all the game objects positions. It also handles collisions for all game objects and results of any interactions with specific game objects.

4. Technical Implementation

A technical implementation I used in my game is the use of trig to handle the tank and bullet movement. I utilized my knowledge of trigonometry to make it so that the tank will rotate when clicking side buttons and only move forward and backward when clicking the forward and backward buttons, and the bullet will be shot in the direction of the tank. I had to first do the calculations for each quadrant and then implement it into my game to make sure that the bullet travels in the same direction as the tank.

5. Most Challenging Part

The most challenging part of the project was how to actually implement the power ups. I has trouble figuring out which classes would handle the power ups and how to modify previous code relating to shooting in order to implement the power ups.

6. Future Fixes and Updates

In the future, I would like to add more unique powerups, including shooting in three different directions, adding a beam that you can shoot and will damage anyone who makes contact with it, etc. I would also like to try and make it multiplayer so that people from different computers can play against each other.
